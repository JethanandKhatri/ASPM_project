ASPM CASE TOOL
Agile Software Project Management
Course Project — Design Prompt for UI/UX
Document Purpose
This document is a detailed design prompt to be shared with a UI designer
to create all screens, flows, and components of the ASPM CASE Tool.
1. Project Overview
The ASPM CASE Tool is a web-based Computer-Aided Software Engineering tool built for Agile Software Project Management. It helps a Project Manager and their team to:
Set up and manage multiple software projects
Enter and store project requirements and features
Estimate project size, effort, cost, and schedule using 6 industry techniques
Manage and prioritize project risks with full RMMM planning
View history of all past projects, estimation runs, and risk logs
Use pre-loaded realistic sample data so the tool is never empty
The tool is for a Project Manager + Team setup. The PM has full control; team members can view and contribute task estimates.

4. Screen-by-Screen Design Requirements
Screen 1 — Login / Register
Two tabs: Login and Register
Login fields: Email, Password, Login button, Forgot password link
Register fields: Full Name, Email, Password, Role selector (Project Manager / Team Member)
Clean centered card layout, logo/tool name at top
No sidebar on this screen
Screen 2 — Dashboard (All Projects)
Top navbar with: tool logo/name, search bar, notification bell, user avatar + name
Left sidebar with: Dashboard, Projects, History, Risk Logs, Settings
Main area shows project cards in a grid (3 per row)
Each project card shows: Project name, domain/type badge, team size, deadline, status badge (Active / Completed), quick stats (features count, risks count)
Top-right button: + New Project
Filter bar: All / Active / Completed, sort by Date / Name / Risk Level
Pre-loaded with 3 sample projects (E-Commerce Website, Hospital Management System, Mobile Banking App)
Screen 3 — Create / Edit Project
Full-page form layout
Fields: Project Name, Description (textarea), Domain (dropdown: Web / Mobile / Desktop / Embedded / Other), Start Date, Deadline, Team Size
Section below: Add Features / Requirements
Each feature row has: Feature name (text), Description (text), Priority (High/Medium/Low dropdown)
+ Add Feature button to add more rows
Remove button per row
Save Project button (primary), Cancel button
Screen 4 — Project Detail Page
Header: Project name, domain badge, status badge, deadline, team size
4 summary metric cards: Total Features, Estimation Status, Total Risks, Project Duration
Tabs below: Overview | Estimation | Risks | Team
Overview tab: project description, features list table (name, priority, status)
Estimation tab: shows all estimation runs done for this project, with version labels (v1, v2) and technique used
Risks tab: summary table of top risks by Risk Exposure
Team tab: list of team members with role badges
Top-right action buttons: Run Estimation, Add Risk, Edit Project
Screen 5 — Estimation Technique Selector
Appears after clicking Run Estimation from project detail
Page title: Select Estimation Technique
6 technique cards in a 2x3 grid, each showing:
Technique name and icon
One-line description of when to use it
Select button
Techniques: Fuzzy Logic, Analogy, Expert Judgment, Decomposition + SD, Story Points, T-Shirt Sizing
User manually picks one technique to proceed
Screen 6 — Estimation: Fuzzy Logic
Left panel: Set average LOC per size category (Very Small / Small / Medium / Large / Very Large)
Main table: list of project features pulled from Screen 3, PM classifies each into a size category via dropdown
Right panel (live): running total LOC estimate, feature count per category (bar chart or counts)
Bottom: Save Estimation button (saves as v1 or next version)
Screen 7 — Estimation: Analogy
Step 1: Select a past project to use as analogy base (dropdown list of completed projects from history)
Past project data auto-fills: LOC, Effort (staff months), Cost ($), Duration (months)
Step 2: Enter new project estimated LOC and size adjustment factor (%)
Results panel: Scaled effort, scaled cost, scaled duration shown instantly
Step 3: Notes field for assumptions/rationale
Save Estimation button
Screen 8 — Estimation: Expert Judgment
PM or team adds tasks manually
Each row: Task Name, Best Case (days), Most Likely (days), Worst Case (days)
Each row auto-calculates Expected Case using PERT formula: (B + 4M + W) / 6
Bottom summary: Total Expected Effort, Total Best, Total Worst, Simple Standard Deviation
+ Add Task row button, remove per row
Save Estimation button
Screen 9 — Estimation: Decomposition + Standard Deviation
Task table: Task Name, Best Case, Worst Case
Auto-detects number of tasks and switches formula:
10 or fewer tasks: Simple SD = (Sum Worst − Sum Best) / 6
More than 10 tasks: Complex SD = square root of sum of individual SD squared
Results: Sum Best, Sum Worst, SD value, 50% confidence range
Info tooltip explaining which formula is being used and why
Save Estimation button
Screen 10 — Estimation: Story Points & T-Shirt Sizing
Two panels side by side:
Left — Story Points: Enter velocity (SP done in iteration 1), total backlog SP, iteration length (weeks). Output: iterations needed, total schedule in weeks.
Right — T-Shirt Sizing: Features table with Size (XS/S/M/L/XL) and Business Value (1-10). Rank by Net Business Value button sorts features and recommends Include / Consider dropping.
Save Estimation button saves both panels as one result

4A. New Feature Screens
Five additional modules added on top of the core tool: Analytics Dashboard, Notifications, Team Collaboration, Gantt Chart, and Reports/Export.
Screen 16 — Analytics Dashboard
Accessible from sidebar > Analytics (separate from per-project dashboard)
Top filter: select project or &apos;All Projects&apos; view
4 chart widgets in a 2x2 grid:
Estimation Accuracy — bar chart comparing estimated vs actual effort/cost per project
Risk Exposure Trend — line chart of total risk exposure over time as risks get resolved
Technique Usage — donut chart showing which estimation technique was used most across projects, with historical accuracy % per technique
Feature Priority Breakdown — pie chart of High/Medium/Low priority features per project
Each widget has a small &apos;view details&apos; link that expands into a larger modal chart
Top summary strip: Total Projects, Avg Estimation Accuracy %, Total Active Risks, On-Track vs Delayed projects count
Screen 17 — Notifications Panel
Bell icon in top navbar opens a dropdown panel (not a full page)
List of notifications, newest first, each with: icon (deadline/risk/estimation type), message, timestamp, unread dot indicator
Notification types to support:
Deadline reminder — e.g. &apos;Hospital Management System deadline in 7 days&apos;
Risk alert — e.g. &apos;New High priority risk added to Mobile Banking App&apos;
Estimation reminder — e.g. &apos;E-Commerce Website has no estimation run yet&apos;
Task assignment — e.g. &apos;Bilal assigned you to Patient Registration module&apos;
Comment mention — e.g. &apos;Jethanand mentioned you in a comment&apos;
Mark all as read button at top of panel
Clicking a notification navigates to the relevant screen (project, risk, or task)
Settings link at bottom: &apos;Manage notification preferences&apos; → goes to Screen 15
Screen 18 — Project Comments / Activity Feed
New tab inside Project Detail Page (Screen 4): Activity
Two sections combined in this tab:
Comments — threaded comments that can be attached to: the whole project, a specific feature, a specific risk, or an estimation run. Each comment shows author avatar, name, timestamp, text, and reply option.
Activity Log — auto-generated feed below comments showing system events, e.g. &apos;Bilal updated Risk #3 status to Resolved&apos;, &apos;Jethanand saved Estimation v2 (Story Points)&apos;, &apos;Muquaddas added 2 new features&apos;
Comment input box at top with @mention support (typing @ shows team member list to tag)
Filter activity feed by: Comments only / System events only / All
Screen 19 — Task Assignment Board
New tab inside Project Detail Page: Tasks (board view)
Kanban-style board with 3 columns: To Do, In Progress, Done
Each task card shows: feature/task name, assigned team member avatar, priority badge, due date
Drag-and-drop cards between columns to update status
+ Add Task button opens a small modal: Task name, Assign to (dropdown of team members), Priority, Due date, Linked feature (optional, pulls from project features list)
Top of board: filter by assignee, filter by priority
Clicking a card opens task detail panel on the right with full description and comment thread
Screen 20 — Gantt Chart / Timeline View
New tab inside Project Detail Page: Timeline
Horizontal Gantt chart: left column lists features/tasks, right area shows horizontal bars across a date axis (weeks/months)
Bar duration is pulled automatically from estimation results (e.g. Story Points iteration schedule or Expert Judgment task days)
Bars are color-coded by status: Not Started (gray), In Progress (blue), Done (green), Overdue (red)
Today marker — vertical line showing current date on the timeline
Zoom control: Week / Month / Quarter view toggle
Hovering a bar shows tooltip: task name, assignee, start date, end date, % complete
Screen 21 — Reports / Export Center
Accessible from sidebar > Reports, and also from a button on Project Detail Page
Select project (or &apos;All Projects&apos; for portfolio report)
Report type checkboxes: Project Overview, Requirements/Features List, Estimation Results (all versions), Risk Table with RMMM, Activity Log
Export format selector: PDF or Excel
Preview panel on the right shows a live preview of the report layout as sections are toggled
Generate Report button — produces downloadable file
Recent Exports list at the bottom: filename, date generated, re-download link

8. Notes for the Designer
Design all 21 screens listed in Section 3 (15 core screens + 5 new feature modules across 6 additional screens)
Use the pre-loaded sample data from Section 5 to populate the screens — do not show empty states
Every estimation screen must have a Save button and show a live result panel as data is entered
The risk table must show the RMMM expand section when a row is clicked
The sidebar must highlight the currently active section, and must include links to Analytics and Reports
The History section must allow a past project to be opened as read-only and used as an analogy base
The dashboard must show project cards with real data from the 3 pre-loaded projects
The Gantt chart and Task Board should use the same pre-loaded features/tasks so data stays consistent across tabs
The Notifications panel should be pre-loaded with 4-5 sample notifications of different types
Design for desktop (1280px+ wide) as the primary viewport
Deliver designs as Figma file or screen-by-screen images
